Definitions and Terminology

A tree TTT﻿ is made up of a set of nodes endowed with a parent-child relationship with the following properties:

  • If TTT﻿ is non-empty, it has a special node called the root that has no parent.
  • Every node vvv﻿ of TTT﻿ other than the root has a unique parent.
  • Following the parent relation always leads to the root (.i.e., the parent-child relationship does not have “cycles”).

Terminologies

Root: node without parent

Internal node: node with at least one child

External/leaf node: node without children

Ancestors: parent and the ancestors of its parent

Descendants: children and the descendants of all its children

Siblings: two nodes with the same parent

Depth (of a node): number of ancestors (not including itself)

Level: the set of nodes at a given depth

Height (of a tree): maximum depth

Subtree: tree made up of some node and its descendants

Edge: pair of nodes (u,v)(u, v)(u,v)﻿ such that one is the parent of the other

Path: a sequence of nodes <v1,v2,…,vn><v_1, v_2, \dots, v_n><v1​,v2​,…,vn​>﻿ such that 2 consecutive nods in the sequence have an edge

Facts

If node xxx﻿ is an ancestor of node yyy﻿, then yyy﻿ is a descendant of xxx﻿.

Ancestor/descendent relations are transitive.

Every node is a descendant of the root.

There may be two nodes where neither is an ancestor (or descendent) of the other.

Every pair of nodes has at least one common ancestor.

The lowest common ancestor (LCA) of xxx﻿ and yyy﻿ is the ancestor of xxx﻿ and yyy﻿ with the greatest depth.

Binary Trees

A binary tree is an ordered tree with the following properties:

  • Each node has at most two children.
  • Each child node is labeled as either the left child or the right child.
  • The ordering is the left child node followed by the right child node.

The right/left subtree is the subtree of which the right/left child is the root.

We say the tree is proper if every internal node has exactly two children.

Applications

  • Arithmetic expression trees
    • Internal nodes store operators
    • External nodes store operands
  • Decision trees
    • Internal nodes store questions
    • External nodes store decisions

Traversing trees

A traversal visits the nodes of a tree in a systematic manner.

When traversing a simpler structure like a list there is one natural traversal strategy (forward or backward).

Trees are more complex and admit more than one natural way:

  • Pre-order Process all nodes of a tree by processing the root, then recursively processing all subtrees.
  • Post-order Process all nodes of a tree by recursively processing all subtrees, then finally processing the root.
  • In-order (only for binary trees)

    Process all nodes of a tree by recursively processing the left subtree, then processing the root, and finally the right subtree.

  • Euler tour traversal

    Walk around the tree, keeping the tree on your left, and visit each node three times: on the left, from below, and on the right.

  • If you pick the first occurrence of each node in an Euler tour traversal, you get the result of a pre-order traversal.
  • If you pick the second occurrence of each node in an Euler tour traversal, you get the result of an in-order traversal.
  • If you pick the last occurrence of each node in an Euler tour traversal, you get the result of a post-order traversal.
