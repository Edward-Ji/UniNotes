Minimum Spanning Trees

Given a graph G=(V,E)G=(V, E)G=(V,E)﻿ with real-valued edge weights wew_ewe​﻿, a minimum spanning tree is a subgraph of GGG﻿ with the weighted sum of all edges minimized.

Applications

  • Network design: telephone, electrical, hydraulic, cables, computer, and roads.
  • Approximation algorithms for NP-hard problems such as traveling salesperson problem, and Steiner tree.
  • Max bottleneck paths.
  • LDPC codes for error correction.

Let S⊆VS \subseteq VS⊆V﻿.

Kruskal’s Algorithm

Kruskal’s algorithm finds a minimum spanning forest of an undirected edge-weighted graph using the cycle property.

First, we sort all edges in ascending order of their weight and create an empty set TTT﻿.

Then, for each edge eee﻿,

  • If adding eee﻿ to TTT﻿ creates a cycle, we discard eee﻿ according to the cycle property;
  • Otherwise, we insert eee﻿ into TTT﻿ until TTT﻿ spans the graph.

To test if an edge (u,v)(u, v)(u,v)﻿ creates a cycle, we use a union-find data structure. We know it creates a cycle if u,vu, vu,v﻿ belongs to the same subset. We unify the subset they are in when we add it to the tree.

Time complexity

The algorithm runs in O(n2)O(n^2)O(n2)﻿ with the simplest union-find data structure.

It runs in O(nlog⁡n)O(n \log n)O(nlogn)﻿ time with the more efficient union-find data structure that keeps track of the cardinality of sets.

It runs in O(n)O(n)O(n)﻿ time with the even more efficient union-find data structure that uses a tree.

Union-Find Data Structure

To efficiently check for cycles, we use the union-find data structure.

The union-find data structure (or the disjoint-set data structure) stores a partition of a set into disjoint subsets. It should support the following operations:

  • Finding the id of the set that contains an element.
  • Unifying the subsets to which two given elements belong.

Implementation and time complexity

Typically, with a union-find data structure with nnn﻿ elements,

  1. We can simply implement with an array that maps from the element (assuming an element can be used as an index) to their subset id. The unifying operation takes O(n)O(n)O(n)﻿ time.
  1. We can keep track of the cardinality of each set. When taking the union of two sets, we change the one that is smaller. The unifying operation takes O(log⁡n)O(\log n)O(logn)﻿ amortized time.
  1. We can keep track of sets using trees. We represent every element as a node and keep track of its parent. While finding any element’s subset id, we also flatten the tree by adding all nodes on the path as direct children of the root node.
