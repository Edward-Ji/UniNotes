Key Concepts

An abstract data type (ADT) is a type defined in terms of its data items and associated operations, not its implementation. It is a specification of the desired behavior from the point of view of the user of the data.

  • Code is easier to understand if different functionalities are separated.
  • Many different systems can use the same library.
  • There can be choices of implementations with different performance tradeoffs, and the client doesn't need to be rewritten extensively.
  • Clients using the ADT need to learn the interface only once.

There are challenges in defining ADTs:

  • Specify how to deal with the boundary cases.
  • Is it possible to reuse existing ADTs?

A data structure is a concrete representation of data. This is from the point of view of an implementer, instead of a user.

A data structure implementation is a class that inherits from the ABC, provides code for all the required methods, and has a constructor. Client code can instantiate from the data structure class and call methods on the instances.

Dynamic-Array-Based Lists

To address the capacity limitation in a list implemented by a fixed-sized array, we can use a dynamic array. When we run out of space, we allocate a larger array and copy existing elements to the new array. Python list is implemented this way. add operation could be slower if the reallocation is required, but still within linear time.

Advantage

  • The space used is O(n)O(n)O(n)Ôªø, also allowing expansion

Iterators

Iterators abstract the process of stepping through a collection of elements one at a time by extending the concept of position. Implemented by maintaining a cursor to the current element. Two notions of iterators are:

  • Snapshot freezes the contents of the data structure (unpredictable behavior if we modify the collection)
  • Dynamic follows changes to the data structure (predictable behavior if we modify the collection)

Stacks and Queues

These ADTs are restricted forms of lists, where insertions and removals happen only in particular locations:

  • Stacks follow last-in-first-out (LIFO)
  • Queues follow first-in-first-out (FIFO)

When do we need to use a less general ADT?

  • Operation names are part of computing culture
  • Specific applications
  • Simpler or more efficient implementations

Stack

Main operations are:

  • push(e) inserts an element e
  • pop() removes and returns the last inserted element

Auxiliary stack operations:

  • top() returns the last inserted element without removing
  • size()
  • isEmpty()

Applications

  • Chain of method calls in computer languages
  • Keep track of a history that allows undoing
  • Context-free grammars
  • Indirectly used as an auxiliary data structure for algorithms or other data structures

Example: an algorithm for parentheses matching

Each (, [, { must be paired with a matching ), ], }. Check if a given string is valid.

The algorithm works as follows: we scan the input string from left to right. If we see an opening character, push it to a stack, otherwise, pop a character from the stack and check if they match. If they match, we proceed, otherwise, we return false.
