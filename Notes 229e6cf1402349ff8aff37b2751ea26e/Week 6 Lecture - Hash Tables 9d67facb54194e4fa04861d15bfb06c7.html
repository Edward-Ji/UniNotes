Handling Collisions

Collisions occur when two or more elements are hashed to the same location. A good hash function makes collisions scarce. However, collisions could still happen and we need to have a method for dealing with them.

Separate Chaining

Let each cell in the map point to a linked list holding the key-value pairs.

All operations are delegated to a list by the hash function. We then operate on that list.

This method is simple but requires additional memory outside the map.

Linear Probing

Concepts

Open addressing, or closed hashing: the colliding item is placed in a different cell of the table.

Linear probing: handles collisions by continuously stepping to the next (circularly) available cell.

Probe: each table cell that is inspected.

Operations

To get a key kkkÔªø in a hash table with linear probing, we start at cell h(k)h(k)h(k)Ôªø and probe consecutive locations until one of the following occurs:

  • An empty cell is found or NNNÔªø cells have been probed (the key kkkÔªø does not exist).

We need to mark the cell defunct when removing keys from the hash table. We replace the removed key to distinguish them from usual empty cells.
