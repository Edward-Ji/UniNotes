Restricted Keys

A simple implementation of a map that uses integer keys as addresses to get items.

Consider a restricted setting in which a map with nnn﻿ items with keys in a range from 000﻿ to N−1N-1N−1﻿, for some N≥nN \ge nN≥n﻿.

  • We use an array of size NNN﻿
  • The Key is just the index of the array

Advantage

  • O(1)O(1)O(1)﻿ operations for get, put, and remove.

Disadvantage

  • Unable to handle generic keys like strings.
  • Poor space complexity and inflexibility, usually N≫nN \gg nN≫n﻿.

Hash Functions

To get around these issues, we use a hash function hhh﻿ to map keys to corresponding indices in an array AAA﻿.

  • hhh﻿ is a mathematical function
  • hhh﻿ is fairly efficient, close to O(1)O(1)O(1)﻿.

Mathematically, a hash function h:K→[0,N−1]h: K \to [0,N-1]h:K→[0,N−1]﻿ maps keys of a given set KKK﻿ to integers in a fixed interval [0,N−1][0, N-1][0,N−1]﻿, where NNN﻿ is the size of the array.

Ideally, a value vvv﻿ of key-value pair (k,v)(k, v)(k,v)﻿ is stored at A[h(k)]A[h(k)]A[h(k)]﻿.

Choice of Hash Functions

A hash function hhh﻿ is sometimes a composition of two function h(k)=f∘g(k)h(k) = f \circ g(k)h(k)=f∘g(k)﻿, where f:K→Zf: K \to \Zf:K→Z﻿ and g:Z→[0,N−1]g: \Z \to [0, N-1]g:Z→[0,N−1]﻿.

The general approach to hash function:

  • View a key of non-aggregate type (e.g. IP address and ID number) as a non-negative integer.
  • View a key of aggregated type (e.g. string and list) as a tuple of integers.

The main goal of a hash function is to “disperse” the keys in a random way. In general, we want to avoid having many keys being hashed to the same location.

Handling Collisions

Collisions occur when two or more elements are hashed to the same location. A good hash function makes collisions scarce. However, collisions could still happen and we need to have a method for dealing with them.

Separate Chaining

Let each cell in the map point to a linked list holding the key-value pairs.

All operations are delegated to a list by the hash function. We then operate on that list.

This method is simple but requires additional memory outside the map.

Performance

Assume that the hash function h:K→[0,N−1]h: K \to [0, N-1]h:K→[0,N−1]﻿ maps nnn﻿ keys.

Let XXX﻿ be a random variable that represents the number of items that map to a bucket in the array AAA﻿, then E(X)=nN\text{E}(X) = \frac{n}{N}E(X)=Nn​﻿.

The parameter E(X)\text{E}(X)E(X)﻿ is called the load factor of the hash table, usually written as α\alphaα﻿.

In the worst case, the time complexity of any operation is O(n)O(n)O(n)﻿. This happens when all the keys collide into a single bucket.

The expected time for hash table operations is O(1+α)O(1+\alpha)O(1+α)﻿.

Applications

Java HashSet uses this method with α<0.75\alpha \lt 0.75α<0.75﻿ and switches from a linked list to a binary search tree if any bucket gets too large.

Linear Probing

Concepts

Open addressing, or closed hashing: the colliding item is placed in a different cell of the table.

Linear probing: handles collisions by continuously stepping to the next (circularly) available cell.

Probe: each table cell that is inspected.

Operations

To get a key kkk﻿ in a hash table with linear probing, we start at cell h(k)h(k)h(k)﻿ and probe consecutive locations until one of the following occurs:

  • The key kkk﻿ is found, return the value;
  • An empty cell is found or NNN﻿ cells have been probed (the key kkk﻿ does not exist).

We need to mark the cell defunct when removing keys from the hash table. We replace the removed key to distinguish them from usual empty cells.

Performance

In the worst case, all operations take O(n)O(n)O(n)﻿ time.

Assuming hash values are uniformly distributed, the expected number of probes for get and put operations is 11−α\frac{1}{1-\alpha}1−α1​﻿ where α=nN\alpha = \frac{n}{N}α=Nn​﻿.

Thus, if the load factor α≪1\alpha \ll 1α≪1﻿, the expected running time for get and put operations is O(1)O(1)O(1)﻿.

Applications

Python dict uses open addressing with α<0.66\alpha < 0.66α<0.66﻿.
