Priority Queue ADT

A special type of ADT map to store a collection of key-value items where we can only remove the smallest key:

size()Return how many items are stored
isEmpty()Return if the queue is empty
insert(k, v)Insert item with key kkk﻿ and value vvv﻿
removeMin()Remove and return the item with the smallest key
min()Return item with the smallest key
remove(k)Remove key kkk﻿ from the priority queue
replaceKey(k1, k2)Replace the key k1k_1k1​﻿ with k2k_2k2​﻿
replaceValue(k, v)Replace the value of key kkk﻿ with vvv﻿

Alternatively, we can have an ADT with the maximum version of these functions, but not both.

Application

A reliable matching engine for modern stock trading systems. Buy and sell orders are organized according to a price-time priority, where price has the highest priority and time is used to break ties. A matching engine can be implemented with two priority queues, one for buy orders and one for sell orders.

Implementation

Sequence-Based

Two types of list implementations:

  • Unsorted list implementation
    • insert(k, v) in O(1)O(1)O(1)﻿ time since we just insert at the beginning or end of the sequence.
    • removeMin() and min() takes O(n)O(n)O(n)﻿ time since we have to traverse the entire list to find the smallest key.
  • Sorted list implementation
    • insert(k, v) in O(n)O(n)O(n)﻿ time since we traverse through the list to find the right place to insert so that the list is still in order.
    • removeMin() and min() takes O(1)O(1)O(1)﻿ time since the smallest key is always at one end of the list.

Priority Queue Sorting

We can use a priority queue to sort a list of keys:

  • Iteratively insert keys into an empty priority queue;
  • Iteratively remove the minimum key to get values in order.

Using a sequence-based implementation of priority queues, the sorting takes O(n2)O(n^2)O(n2)﻿ time.

Note the correspondence between them and two common sorting algorithms:

  • sorting with unsorted-list-based priority queue & selection sort
    • insertion → O(n)O(n)O(n)﻿
    • removal/selection → O(n2)O(n^2)O(n2)﻿
  • sorting with sorted-list-based priority queue & insertion sort
    • insertion → O(n2)O(n^2)O(n2)﻿
    • removal/selection → O(n)O(n)O(n)﻿

Operations

Insertion

The operation has O(log⁡n)O(\log n)O(logn)﻿ time complexity.

  • Create a new node with the given key.
  • Find the location for the new node.

    Starting from the last node, go up until we reach a left child node or the root. If we reach the root, we go to the sibling (the right child of the parent). Either way, we continuously go to the left child until we reach a leaf node. We insert the new node as the left child of that leaf node.

  • Restore the heap-order property.

    Continuously swapping the node with its parents until its parent does not have a larger key.

Removal

The operation has O(log⁡n)O(\log n)O(logn)﻿ time complexity.

  • Finding the future last node.

    Starting from the current last node, go up until we reach a right child node or the root. If we do not reach the root, we go to the sibling (the left child of the parent). Either way, we continuously go to the right child until we reach a leaf node. We insert the new node as the left child of that leaf node.

  • Replace the root node with the last node.
  • Restore the heap-order property.

    Continuously swapping the node with its smaller children until neither child has a smaller key.

Other Implementation Details

Entry: a key-value pair.

Comparator: a function that takes in two keys a,ba, ba,b﻿ and return

  • a negative integer if a<ba \lt ba<b﻿;
  • zero if a=ba = ba=b﻿;
  • a positive integer if a>ba \gt ba>b﻿.

Applications

An online trading system uses two priority queues to store orders as (p,t,s)(p, t, s)(p,t,s)﻿ entries, where

  • The key is (p,t)(p, t)(p,t)﻿ - the price ppp﻿ , and the time ttt﻿. We first sort by ppp﻿, then break ties with ttt﻿.
  • The value is sss﻿, the number of shares in the order.
