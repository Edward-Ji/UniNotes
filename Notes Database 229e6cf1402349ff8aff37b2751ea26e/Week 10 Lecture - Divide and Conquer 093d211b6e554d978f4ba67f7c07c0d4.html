Divide and Conquer

Divide and conquer algorithms can normally be broken into:

  • Divide Solve directly for the base case. For other cases, break up the problem into several parts and recursively solve each sub-problem.
  • Conquer Combine the solutions of each part into the overall solution.

Binary Search

Given a sorted array AAA﻿ with no duplicate elements, we want to find if a particular item xxx﻿ exists in the array.

Naive Algorithm

A naive approach iterates through the entire array and checks if the item equals our target. This algorithm runs in O(n)O(n)O(n)﻿ time.

Binary Search Algorithm

  • If the array is empty, then return false.
  • Compare xxx﻿ to the middle element, namely the element at index ⌊n2⌋\left\lfloor \frac{n}{2}\right\rfloor⌊2n​⌋﻿ assuming the index starts at 000﻿.
    • If that middle element is equal to xxx﻿, return true.
    • If that middle element is greater than xxx﻿, recursively search between index 0,⌊n2⌋0, \left\lfloor \frac{n}{2}\right\rfloor0,⌊2n​⌋﻿.
    • If that middle element is smaller than xxx﻿, recursively search between index ⌊n2⌋,n\left\lfloor \frac{n}{2}\right\rfloor, n⌊2n​⌋,n﻿.

Sketch of Proof

The invariant in our algorithm is that xxx﻿ is always in the array we are searching.

The base case is trivially true because if AAA﻿ is empty, clearly the element xxx﻿ is not in the array.

Now, we prove that the algorithm returns true if xxx﻿ exists in the array.

Suppose that the current call stack is searching xxx﻿ in a sorted array AAA﻿ of size nnn﻿. By definition, for any two indices i,j∈[0,n)i, j \in \left[0, n\right)i,j∈[0,n)﻿, i<ji \lt ji<j﻿ if and only if A[i]<A[j]A[i] < A[j]A[i]<A[j]﻿.

If xxx﻿ exists in AAA﻿, we denote the index of xxx﻿ in array AAA﻿ as kkk﻿.

If x=A[⌊n2⌋]x = A\left[\left\lfloor \frac{n}{2}\right\rfloor\right]x=A[⌊2n​⌋]﻿, we return true, the result is trivially true. If x<A[⌊n2⌋]x < A\left[\left\lfloor \frac{n}{2}\right\rfloor\right]x<A[⌊2n​⌋]﻿, then k<⌊n2⌋k < \left\lfloor \frac{n}{2}\right\rfloork<⌊2n​⌋﻿, so the array we passed to the next call stack A′=A[0:⌊n2⌋]A'=A\left[0: \left\lfloor \frac{n}{2}\right\rfloor\right]A′=A[0:⌊2n​⌋]﻿ still contains xxx﻿. Similarly, the algorithm also works if x>A[⌊n2⌋]x > A\left[\left\lfloor \frac{n}{2}\right\rfloor\right]x>A[⌊2n​⌋]﻿.

By mathematical induction, in all call stacks, AAA﻿ always contains the xxx﻿ if such xxx﻿ exists in the original array. Hence, it will return true at the end if such xxx﻿ exists.

Analysis

The most common way to solve the time complexity of a divide-and-conquer algorithm is to use recurrence relations.

For our binary search algorithm, let T(n)T(n)T(n)﻿ be the time complexity for the algorithm on an array AAA﻿ of size nnn﻿. Then, if we implement the subarray with the low and high indices, instead of copying both halves of the array, the cost is O(1)O(1)O(1)﻿ for each call stack.

Linked List Implementation

To locate its middle element in a linked list of size nnn﻿, we need O(n)O(n)O(n)﻿ time. Therefore, if we denote the running time of a linked list implemented binary search as T(n)T(n)T(n)﻿, then T(n)=T(n/2)+O(n)T(n) = T(n/2) + O(n)T(n)=T(n/2)+O(n)﻿. This solves to T(n)=O(n)T(n) = O(n)T(n)=O(n)﻿.

Merge Sort

Given an array AAA﻿ of size nnn﻿. We want to sort it in ascending order.

Algorithm

Recurrence (Divide)

If the array AAA﻿ has one or no elements, we return the array itself. Otherwise, we divide the array into two halves A1A_1A1​﻿ and A2A_2A2​﻿, and recursively sort each half with this algorithm. We get two sorted halves B1B_1B1​﻿ and B2B_2B2​﻿.

Merge (Conquer)

We merge the two sorted halves B1B_1B1​﻿ and B2B_2B2​﻿ of size m1m_1m1​﻿ and m2m_2m2​﻿ respectively in such a way that the final array is still sorted.

  • We create a new array CCC﻿ with the size of m3=m1+m2m_3 = m_1 + m_2m3​=m1​+m2​﻿.
  • We keep track of the smallest element in B1B_1B1​﻿ and B2B_2B2​﻿. We insert the smaller one into CCC﻿.
  • Repeat the previous step until both B1B_1B1​﻿ and B2B_2B2​﻿ are exhausted.

Quick Sort

Given an array AAA﻿ of size nnn﻿. We want to sort it in ascending order.

Algorithm

Divide

We first choose an element from the list as the pivot. We then partition the array into three subarrays that include elements less than, equal to, and greater than the pivot respectively. We recursively sort the two subarrays that do not contain elements equal to the pivot.

Conquer

We join the three subarrays back together.
