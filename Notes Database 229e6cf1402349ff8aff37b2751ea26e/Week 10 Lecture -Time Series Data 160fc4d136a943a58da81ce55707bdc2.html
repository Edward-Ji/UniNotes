Motivation for Temporal Data

Temporal data is ubiquitous. All data can be associated with a time at which we observe the data.

Big data also includes historic data. People are concerned with the trend.

There is limited and non-standardized support for temporal data management in DBMSs.

  • Conventional databases represent data with static snapshots.
  • Management of temporal aspects is implemented by the application. It adds complexity to the programs.
  • SQL only added support for temporal tables in 2011.

Temporal Database Concepts

Time Data Types

SQL supports both instances and intervals of time.

Instance data types

  • SQL-92 data types DATE, TIMESTAMP, TIME
  • Modeled as time instances (or points) with a partial order
  • Implementation is application-specific.

Interval data types

  • For example, some SQL dialects support year-month intervals INTERVAL YEAR TO MONTH.
  • Many DBMS does not support this type of temporal data.

Valid time

  • Valid time (or fact time) records the time when a fact is true in the real world.
  • It can move forward and backward.

Transaction time

  • Transaction time (or database time) records the history of database activity.
  • It can only move forward.

Transaction time databases

  • The type of database retains and provides access to prior states of a database.
  • Transaction time does not have to be reflected explicitly in the relational schema.

Temporal Database

Types of temporal statements: current, sequenced, and non-sequenced.

A semi-temporal database only stores the start date, while a temporal database stores both the start date and end date.

We commonly use the maximum timestamp or the start time to denote “until the present”.

SQL 2011 added support for valid and transaction time as metadata (not a new data type). It is only supported by a few commercial DBMSs.

PostgreSQL has a new data type DATERANGE to represent time periods. It also provides an index on such a data type with generalized inverted search trees.

Time Series Data

Time series data are observations made sequentially overtime at discrete time points.

There are commonly two objectives when working with time series.

  • Time series analysis
    • Time-domain methods
    • Frequency domain methods (.eg. Fourier Transform)
  • Time series forecasting

Workflow

  • Retrieve data
    • Point-based: hard to aggregate.
    • Sequence-based representations: require complex data types (e.g. array) and functions (e.g. array aggregation).
  • Store data: whether to use a dedicated time-series database.
  • Analyze data
  • Visualize and report data
