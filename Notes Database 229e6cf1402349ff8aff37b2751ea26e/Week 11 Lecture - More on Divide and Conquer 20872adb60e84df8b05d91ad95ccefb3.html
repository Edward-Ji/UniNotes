More Problems

Pareto Front

Given a set SSS﻿ of nnn﻿ distinct points in a 2D plane, find the maximum set. A point is considered to be in the maximum set if all other points either have a smaller x-value or a smaller y-value.

Naive Algorithm

We check if it is a point is in the maximum set for each point in the set SSS﻿ by iterating through the rest of the set and comparing their xxx﻿ and yyy﻿ values.

Two nested loops iterating through the whole set (and the whole set minus one point) give us O(n2)O(n^2)O(n2)﻿ complexity.

Algorithm

Prepare

We first sort all points by their xxx﻿ coordinate, then by their yyy﻿ coordinate if their xxx﻿ coordinates are the same in one go.

Divide

If our set has zero or one point, return the set itself. Otherwise, we split the array into two halves, and recursively compute the maximum set of each half. We denote the maximum set for the left half and the right half as AAA﻿ and BBB﻿ respectively.

Conquer

We merge AAA﻿ and BBB﻿ together:

  1. Find the highest point p∈Bp \in Bp∈B﻿.
  1. Compare every point a∈Aa \in Aa∈A﻿, if a.y>p.ya.y > p.ya.y>p.y﻿, add qqq﻿ to the merged maximum set.
  1. Add every point b∈Bb \in Bb∈B﻿ to the merged maximum set.

Analysis

The preprocessing step involves sorting only. The most efficient sorting algorithms run in O(nlog⁡n)O(n \log n)O(nlogn)﻿ time. So the preprocessing step runs in O(nlog⁡n)O(n \log n)O(nlogn)﻿.

The divide step splits the array into two halves. Depending on whether we create a copy of the array or just a view, the running time is O(n)O(n)O(n)﻿ or O(1)O(1)O(1)﻿. Not that it matters in asymptotic analysis.

If we denote the running time of our algorithm without the preprocessing step with a set of nnn﻿ points as T(n)T(n)T(n)﻿. The step where we recursively compute the maximum sets of both halves runs in 2T(n2)2T(\frac{n}{2})2T(2n​)﻿ time.

The final conquer step runs in O(n)O(n)O(n)﻿ time.

T(n)=2T(n2)+O(n)T(n) = 2T(\frac{n}{2}) + O(n)T(n)=2T(2n​)+O(n)﻿ solves to T(n)=O(nlog⁡n)T(n) = O(n\log n)T(n)=O(nlogn)﻿.

So, the overall running time of our algorithm including the preprocessing step is O(nlog⁡n+nlog⁡n)=O(nlog⁡n)O(n\log n + n\log n) = O(n \log n)O(nlogn+nlogn)=O(nlogn)﻿.
