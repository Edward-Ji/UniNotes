Concepts

A graph GGG﻿ is a pair (V,E)(V, E)(V,E)﻿, where VVV﻿ is a set of nodes, called vertices, and EEE﻿ is a collection of pairs of vertices, called edges.

There are two edge types:

  • Directed edge: ordered pair of vertices (u,v)(u, v)(u,v)﻿, where uuu﻿ is the origin/tail, and vvv﻿ is the destination head.
  • Undirected edge: unordered pair of vertices (u,v)(u, v)(u,v)﻿.

Applications

  • Electronic circuits
  • Transportation networks
  • Computer networks
  • Entity-relationship diagram

Terminology

Undirected Graphs

  • Edges connect endpoints.
  • A vertex is incident with an edge if the vertex is one of the endpoints of that edge, or we say that edge is incident to the vertex.
  • Adjacent vertices have the same endpoint.
  • The degree is the number of edges on a vertex.
  • Parallel edges share the same endpoints.
  • Self-loop has the same vertex for its endpoints.
  • Simple undirected graphs have no parallel edges or self-loops.

Directed Graphs

  • Edges go from tail to head.
  • Out-degree is the number of edges of which the vertex is the tail.
  • In-degree is the number of edges of which the vertex is the head.
  • Multiple edges (or Parallel edges) share the same head and tail.
  • Simple directed graphs have no multiple edges or self-loops but can have “anti-parallel” loops.

Paths

  • A path is a sequence of edges which joins a sequence of vertices.

    A simple path is one where all vertices are distinct.

  • A cycle is a path that starts and ends at the same vertex.

    A simple cycle is one where all vertices are distinct.

Subgraphs

Let G=(V,E)G = (V, E)G=(V,E)﻿ be a graph. We say S=(U,F)S = (U, F)S=(U,F)﻿ is a subgraph of GGG﻿ if U⊆VU \subseteq VU⊆V﻿and F⊆EF \subseteq EF⊆E﻿.

Connectivity

A graph G=(V,E)G = (V, E)G=(V,E)﻿ is connected if there is a path between every pair of vertices in VVV﻿.

A component of a graph GGG﻿ is a connected subgraph that is not part of any larger connected subgraph.

Trees

An acyclic graph has no cycles. A connected acyclic graph is known as a tree, and a possibly disconnected acyclic graph is known as a forest (i.e., a collection of trees).

An unrooted tree TTT﻿ is a graph such that TTT﻿ is connected and has no cycles.

Every tree on nnn﻿ vertices has n−1n-1n−1﻿ edges.

A spanning tree is a connected subgraph on the same vertex set. A graph has more than one spanning tree unless the graph is a tree.

Given any vertex vvv﻿, deg⁡(v)\deg(v)deg(v)﻿ gives the degree of vvv﻿.

Implementations

Edge List

An implementation of a graph is an edge list structure, which consists of a vertex sequence and an edge sequence.

Each edge object points to two vertices that represent its endpoints.

Adjacency List

In addition to an edge list, each vertex keeps a sequence of edges incident to it.

Adjacency Matrix

Suppose we have nnn﻿ vertices, we construct an nnn﻿ by nnn﻿ matrix.

Graph Traversals

A traversal of a graph is a systematic procedure for examining all its vertices and edges.

The following are two traversal methods for undirected graphs.

Depth-First Search

In a depth-first search, we follow outgoing edges leading to unvisited vertices whenever possible and backtrack if there is none.

If an edge leads to an unvisited vertex, we call it a DFS edge; otherwise, we call it a back edge.

Performance

If we use an adjacency list implementation, the depth-first search runs in O(m+n)O(m+n)O(m+n)﻿ time.

    The looping of all vertices takes O(n)O(n)O(n)﻿ time.

    To find the adjacent vertices recursively for each vertex vvv﻿, the time complexity is O(deg⁡v)O(\deg v)O(degv)﻿.

    The overall time of the recursive calls to visiting all nodes is O(∑v∈Vdeg⁡v)=O(m)O\big(\sum_{v\in V}\deg v\big)=O(m)O(∑v∈V​degv)=O(m)﻿.

    The overall time of the traversal is O(m+n)O(m+n)O(m+n)﻿.

If we use an adjacency matrix implementation, it runs in O(n2)O(n^2)O(n2)﻿ time.

Breadth-First Search

In a breadth-first search, we visit all vertices closer to the start vertex before visiting vertices that are further.

The distance between two vertices is determined by the minimum number of edges in all paths between them.

  • The breadth-first search visits all vertices in VCV_CVC​﻿.
  • The breadth-first search forms a tree. If we denote the parent of any vertex vvv﻿ as parent v\text{parent}\,vparentv﻿ in such tree,

BFS can be used to solve other graph problems in O(m+n)O(m + n)O(m+n)﻿ time:

  • Test whether a graph is connected.
  • Test if a graph is bipartite (more on this next week).

For directed graphs, we only consider adjacent vertices with outgoing edges.

Topological Sort

Directed acyclic graphs (DAG) is a directed graph that has no directed cycles.

Every DAG can be topologically sorted. In other words, vertices can be laid out from left to right such that all edges go left to right as well.

BFS-based implementation

We first run DFS on the graph G=(V,E)G = (V, E)G=(V,E)﻿. The graph is not acyclic if there exists an edge e∈Ee \in Ee∈E﻿ such that eee﻿ is a back edge in DFS. Otherwise, the result is VVV﻿ in reverse order of DFS-visit call return.

Performance

Same as DFS, the algorithm runs in O(m+n)O(m+n)O(m+n)﻿ time.

Strongly Connected Components

Let G=(V,E)G = (V, E)G=(V,E)﻿ be a directed graph. A strongly connected component (SCC) of GGG﻿ is a graph C=(VC,EC)C = (V_C, E_C)C=(VC​,EC​)﻿ of vertices such that

  • for any u,v∈VC⊆Vu, v \in V_C \subseteq Vu,v∈VC​⊆V﻿, there is a path from uuu﻿ to vvv﻿ in GGG﻿.
  • no superset of VCV_CVC​﻿ has the above property.

Implementation

First, we perform a DFS on a given graph G=(V,E)G = (V, E)G=(V,E)﻿.

Then, we construct a graph F=(V,{(v,u)∣(u,v)∈E})F = \Big(V, \big\{(v, u) \mid (u, v) \in E\big\}\Big)F=(V,{(v,u)∣(u,v)∈E})﻿. In other words, FFF﻿ is GGG﻿ with all edges reversed.

We perform DFSes on the vertices in the first DFS in order on the new graph FFF﻿.

The resultant forest is the strongly connected components of GGG﻿.

Performance

It takes O(m+n)O(m + n)O(m+n)﻿ to run.
