Web Scraping

How can we extract data from a website?

  • Scrapping the web (HTML)

Web pages are written in HTML, (HyperText Markup Language). It is the code that is used to structure a web page and its content.

HTML and XML are semistructured data models.

  • They can handle optional attributes and structure variants.
  • They are self-describing and do not require a schema.
  • They support nesting and are represented by a tree.

General Approach

  • Reconnaissance

    Identify the source, and check its structure and content.

  • Web page retrieval

    Download one or more pages from the source.

    Typically in a script or program that auto-generates new URLs based on website structure and its URL format.

  • Data extraction

    Content parsing, raw data extraction.

  • Data cleaning

    Transform into the required format.

  • Data storage

    Analysis, combining with other datasets.

URLs

Uniform Resource Locator

  • “address” format on the web
  • General format protocol://path/to/site/page
  • Can be generated using a script.

Legal

In principle, publicly available data is free for personal usage.

Check copyright, ask, don’t overload.

Some sites come with a robots.txt that instructs which agents are allowed. It is also called a robots exclusion standard/protocol.

HTML

Markup via opening and closing tags.

Predefined in HTML standards.

Interpreted by web browsers for display, or be extracted with scripts.

Selecting Content

  • Text patterns

    Simple, but can not handle complex patterns.

  • DOM navigation

    The Document Object Model (DOM) is a programming interface for web documents. It represents the page so that programs can change the document structure, style, and content. The DOM represents the document as nodes and objects.

  • CSS selectors

    Based on the tag types, class specifications, and IDs elements. Easy to specify, but depends on well-used CSS classes and IDs.

  • XPath expressions

    A powerful language that allows navigation along the document tree and selects all nodes or even sub-trees which match the path expression.
