1.1 Information Is Bits + Context

The program begins as a source program (or source file).

A C program file is just a text file.

All information in the system is represented as a bunch of bits. The only thing distinguishing different data objects is the context in which we view them.

1.2 Programs Are Translated by Other Programs into Different Forms

high-level C program → low-level machine-language instructions

In the form of an executable object program.

The translation is performed by a compiler driver e.g. gcc.

  • Preprocessing phase. Modifies the original C program according to directives that begin with the ‘#’ character. → *.i
  • Compilation phase. The compiler translates the text file *.i into the text file *.s - assembly-language program
  • Assembly phase. The assembler translates *.s into machine-language instructions, packages them in a relocatable object program, and stores the result in the object file *.c. This is binary.
  • Linking phase. The linker handles the merging of separate precompiled object files. For example, printf.o is linked if you use <stdio.h> and printf in your program.

1.3 It Pays to Understand How Compilation Systems Work

  • Optimizing program performance.
  • Understanding link-time errors.
  • Avoiding security holes, like buffer overflow vulnerabilities.

1.4 Processors Read and Interpret Instructions Stored in Memory

1.4.1 Hardware Organization of a System

Buses

Running throughout the system is a collection of electrical conduits called buses that carry bytes of information back and forth between the components. Buses are typically designed to transfer fixed-size chunks of bytes known as words. Most machines have word sizes of either 4 bytes or 8 bytes, but we do not assume a fixed definition of word size.

I/O Devices

Input/output devices are the system’s connection to the external world, e.g. keyboard and mouse for input, display for output, and disk drive for long-term storage. Each I/O device is connected to the I/O bus by either a controller or an adapter.

  • Controllers are chip sets in the device itself or on the system’s main printed circuit board (motherboard).
  • Adapters are cards that plug into a slot on the motherboard.

Main Memory

The main memory is a temporary storage device that holds both a program and the data it manipulates while the processor is executing the program. Physically, the main memory consists of a collection of dynamic random access memory (DRAM) chips. Logically, memory is organized as a linear array of bytes, each with its own unique address (array index) starting at zero.

Processor

The central processing unit (CPU), or simple processor, is the engine that interprets (or executes) instructions stored in the main memory. At its core is a word-size storage device (or register) called the program counter (PC). At any point in time, the PC points at (contain the address of) some machine-language instruction in the main memory.

A process repeatedly executes the instruction pointed at by the program counter and updates the program counter to point to the next instruction. A processor operates according to its instruction set architecture. The instructions revolve around the register file - a small storage device that consists of a collection of word-sized registers - and the arithmetic/logic unit (ALU).

The actual implementation of a processor’s program execution is its microarchitecture.

1.5 Caches Matter

A system spends a lot of time moving information.

machine instructions and data strings @ disk → copied to main memory → copied to processor

Much of this copying is overhead that slows down the “real word” of the program.

Physics dictates that larger storages are slower than smaller ones. The faster ones are more expensive.

The processor-memory gap is increasing. System designers include smaller, faster storage devices than main memory called cache memories (or simply caches) that serve as temporary staging areas for information that the processor is likely to need in the near future. A typical cache has two levels - L1, L2 - implemented with static random access memory (SRAM). Newer and more powerful systems could have three levels. The idea behind caching is locality, the tendency for programs to access data and code in localized regions.
