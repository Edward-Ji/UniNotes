Greedy Algorithms

A greedy algorithm is an approach for solving a problem by selecting the best option available at the moment. It doesn't worry whether the current best result will bring the overall optimal result. The algorithm never reverses the earlier decision even if the choice is wrong. [link]

Interval Partitioning Problem

Given a set SSS﻿ of nnn﻿ lectures. Each lecture iii﻿ starts at sis_isi​﻿ and finishes at fif_ifi​﻿. We want to find the minimum number of classrooms to schedule all lectures so that no two occur at the same time in the same room.

Solution

We first sort the intervals by increasing starting time. Then, we repeatedly assign each lecture to any compatible classroom.

In order to decide on a compatible classroom, we use a priority queue based on the end time of scheduled lectures.

Huffman Coding

Run-length encoding is a form of lossless data compression in which runs of data (sequences in which the same data value occurs in many consecutive data elements) are stored as a single data value and count, rather than as the original run. For example, RRRRRUNNN becomes 5R1U2N in run-length encoding.

Better methods use prefix codes. Huffman coding is a type of prefix code that is commonly used in lossless data compression.

Prefix Code

Problem

Given a string XXX﻿, we want to efficiently encode XXX﻿ into a smaller string YYY﻿.

Solution

Let SSS﻿ be a set of characters in XXX﻿, we compute the frequency f(c)f(c)f(c)﻿ for each character c∈Sc \in Sc∈S﻿.

A code is a mapping of each character of an alphabet to a binary codeword.

A prefix code is a binary code such that no codeword is the prefix of another codeword.

An encoding tree TTT﻿ represents the prefix codes of the set of characters.

  • Each external node stores a character.
  • The codeword of a character is given by the path from the root to the external node storing the character.

Then, we can encode string XXX﻿ into YYY﻿ by mapping each character to its codeword.

Caveats

Although greedy algorithms are easy to design, they are not always optimal. Even if they are, small modifications could render them suboptimal. For example, the algorithm for the general knapsack problem does not work for 0-1 knapsack problems. In the 0-1 knapsack problem, we either take an item entirely or we don’t take any at all.
