Relational Database Concepts

A database approach, as opposed to a file-based approach to data analysis, has the following features:

  • A central repository of shared data
  • Dat is managed by software: a Database Management System (DBMS)
  • Data is accessed by application users through the DBMS

What is a database?

A database is a shared collection of logically related data and its description. The database represents the entities (real-world things), the attributes (their relevant properties), and the logical relationships between the entities.

Advantages of the database approach to data science

  • Data is managed, so quality can be enforced by the DBMS
    • Improved data sharing
    • Enforcement of data quality standards
    • Integrity constraints, data validation rules
    • Declarative data access (SQL query language)
    • Security, backup/recovery, concurrency
  • Program-data independence
    • Metadata stored in DBMS: applications don’t care about data formats
    • Data queries/updates implemented by DBMS

More Advanced SQL

Accessing CSV from PostgreSQL

  • Import CSV data with client-side software
    • pgsql command-line tool: \COPY command
    • pgAdmin GUI: Import... menu
  • Server-side COPY command

    PostgreSQL server provides a COPY statement, but it needs file-system level access to files on the server.

  • SQL INSERT statements

    Typically involves Unix/Python scripting to convert CSV lines into INSERTs.

  • Access external CSV data from PostgreSQL

SQLite

It is a “lite” embedded open-source DBMS.

Key features

  • relational data model with unique “type affinity”
  • SQL
  • sqlite3 or embedded library
  • single database file
  • transaction support

Type affinity

  • Each column has a core data type (INTEGER, REAL, TEXT, BLOB), but no native support for DATE or TIMESTAMP.
  • Dynamic typing per value: date type is stored with each attribute value.
